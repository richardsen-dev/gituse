# gituse
1.stage：暂存区；
2.fork:复刻别人的仓库到自己的仓库中；
3.clone:克隆仓库到本地；
4.尽量不要动master分支，本地开发要基于dev分支进行；
5.还原:回退到上次提交状态丢弃修改部分；
5.推送:远端建有相应分支，拉取到本地后即可推送；
6.合并:本地dev分支修改后切换到master分支，将修改后的dev分支与master分支合并后推送；
7.拉取pull：从远程仓库拉取最新版本到本地，自动合并；即暴力合并，适用仅一人开发；
8.提取/获取fetch：从远程仓库提取最新版本到本地，不自动合并；适合团队协作开发；
9.推送push：将本地仓库上传至远端；
10.同步sync：先拉取再推送
